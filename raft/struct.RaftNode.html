A RaftNode acts as a replicated state machine. The server's role in the cluster depends on it's own status. It will maintain both volatile state (which can be safely lost) and persistent state (which must be carefully stored and kept safe).

Creates a new RaftNode with the neighbors specified. id should be a valid index into nodes. The idea is that you can use the same nodes on all of the clients and only vary id.
